# 습관 트래커 (Habit Tracker) — 설계 문서

> 작성일: 2026-06-29
> 목적: Swift/iOS 학습용 첫 앱. SwiftUI 핵심 개념을 단계적으로 익힌다.
> 작성자 배경: 프론트엔드 개발자, Swift/iOS는 완전 처음, 백엔드 지식 없음.

---

## 1. 개요

매일 반복하는 습관(예: 물 마시기, 운동, 독서)을 등록하고 관리하는 iOS 앱.
서버 없이 **기기 한 대 안에서만** 동작한다 (프론트엔드 비유: 서버 API 없이
브라우저 localStorage만 쓰는 SPA).

이 문서는 전체 로드맵 중 **1단계**에 집중한다. 나머지 단계는 "향후 계획"으로만 기록한다.

### 학습 로드맵 (전체 그림)

| 단계 | 내용 | 새로 배우는 것 |
|------|------|---------------|
| **1단계 (이번 스펙)** | 습관 목록 + 추가/수정/삭제 (메모리 상태만) | SwiftUI 기초, 상태 관리, 리스트, 화면 이동, 입력 폼 |
| 2단계 | 기기에 저장 (껐다 켜도 유지) | 로컬 영속 저장 |
| 3단계 | 오늘 체크 + 연속 달성일(streak) | 날짜 다루기, 파생 상태 |
| 4단계 | 상세 화면 + 달력 + 통계 | 차트, 복합 화면 |
| 5단계 | 매일 알림(리마인더) | 로컬 푸시 알림 |

---

## 2. 1단계 기능 범위

### 포함 (MVP)
1. **습관 목록 보기** — 등록된 습관들이 리스트로 표시된다.
2. **습관 추가** — 이름, 아이콘, 색상을 입력해 새 습관을 만든다.
3. **습관 수정** — 기존 습관의 이름/아이콘/색상을 바꾼다.
4. **습관 삭제** — 목록에서 습관을 지운다.

### 의도적으로 제외 (1단계에서는 안 함)
- **저장(영속성)**: 앱을 완전히 종료하면 데이터가 사라진다. 이는 버그가 아니라
  의도된 동작이다 (새로고침하면 state가 초기화되는 React 앱과 동일). 2단계에서 추가.
- 오늘 체크 / streak / 통계 / 알림 → 3단계 이후.
- 회원가입·로그인·클라우드 동기화·기기 간 공유 → 서버가 필요하므로 범위 외.

## 3. 화면 구성

화면은 2개.

### 화면 1 — 습관 목록 (메인, `HabitListView`)
```
┌─────────────────────────┐
│  내 습관            [ + ]│   ← 우측 상단 + 버튼 → 추가 화면 열기
├─────────────────────────┤
│  💧  물 마시기           │
│  🏃  운동하기            │   ← 각 행 = 아이콘 + 이름
│  📖  독서                │      · 행 탭 → 수정 화면
│                         │      · 행 왼쪽으로 밀기(swipe) → 삭제
│   (습관이 없으면         │
│    "습관을 추가해보세요"  │   ← 빈 상태 안내 문구
│    안내 표시)            │
└─────────────────────────┘
```

### 화면 2 — 습관 추가 / 수정 (`HabitFormView`)
```
┌─────────────────────────┐
│ [취소]   새 습관   [저장]│   ← 수정 모드일 땐 제목 "습관 수정"
├─────────────────────────┤
│  이름                    │
│  ┌───────────────────┐   │
│  │ 운동하기          │   │   ← 텍스트 입력
│  └───────────────────┘   │
│  아이콘  💧 🏃 📖 🍎 💊  │   ← 정해진 후보 중 선택
│  색상   🔴 🟠 🟢 🔵 🟣   │   ← 정해진 후보 중 선택
└─────────────────────────┘
```

- **추가와 수정은 같은 화면(`HabitFormView`)을 재사용**한다.
  - 추가: 빈 값으로 열림. [저장] 시 목록에 새 항목 추가.
  - 수정: 기존 값이 채워진 채로 열림. [저장] 시 해당 항목 갱신.
  - [취소]: 변경 없이 닫힘.
- 프론트엔드 비유: 재사용 가능한 Form 컴포넌트에 `mode`(create/edit)와
  초기값을 prop으로 넘기는 패턴.

## 4. 데이터 모델

습관 하나가 가지는 정보 (객체 1개):

```
Habit (습관) — Identifiable
 ├─ id    : UUID        고유 식별자 (목록에서 각 행을 구분하는 키)
 ├─ name  : String      이름 (예: "운동하기")
 ├─ icon  : String      아이콘 (이모지 또는 SF Symbol 이름)
 └─ color : (색상 표현)  목록/폼에서 쓸 색
```

- 1단계에서는 습관들을 **메모리 안의 배열** `[Habit]` 로만 보관한다.
- 화면은 이 배열을 바라보다가, 배열이 바뀌면 SwiftUI가 자동으로 다시 그린다
  (React state ↔ 화면 관계와 동일).

### 색상 표현 방식
- 자유 색이 아니라 **미리 정해둔 후보 색 팔레트**(예: 빨강/주황/초록/파랑/보라)
  중 하나를 고른다. 초보 단계에서 색 데이터를 단순하게 다루기 위함.
- 모델에는 색을 식별할 수 있는 단순 값(예: enum case 또는 색 이름 문자열)으로 저장하고,
  화면에서 실제 `Color`로 변환한다.

## 5. 상태 관리 / 데이터 흐름

- 습관 배열의 "단일 출처(single source of truth)"는 한 곳에 둔다.
  - 권장: `@Observable` 클래스(예: `HabitStore`)가 `habits: [Habit]` 를 보유하고
    `add / update / delete` 메서드를 제공.
  - 프론트엔드 비유: 작은 전역 스토어(Zustand/Pinia) 하나.
- 화면(`HabitListView`, `HabitFormView`)은 이 스토어를 바라보며 읽고, 메서드로 변경한다.
- 추가/수정 화면은 **시트(sheet, 아래에서 올라오는 모달)** 로 띄운다.

### 흐름 예시
- 추가: 목록 [+] → 빈 폼 시트 → 입력 → [저장] → `store.add(...)` → 시트 닫힘 → 목록 자동 갱신
- 수정: 행 탭 → 해당 habit으로 채워진 폼 시트 → [저장] → `store.update(...)` → 갱신
- 삭제: 행 swipe → `store.delete(...)` → 목록에서 사라짐

## 6. 기술 스택

| 항목 | 선택 | 프론트엔드 비유 |

| 언어 | Swift | JavaScript / TypeScript |

## 7. 파일 구조 (예상)

```
HabitTracker/
 ├─ HabitTrackerApp.swift     앱 진입점 (스토어 생성·주입)
 ├─ Models/
 │   └─ Habit.swift           Habit 모델 + 색 팔레트 정의
 ├─ Store/
 │   └─ HabitStore.swift      습관 배열 보관 + add/update/delete
 └─ Views/
     ├─ HabitListView.swift   메인 목록 화면
     ├─ HabitRowView.swift    목록의 한 행
     └─ HabitFormView.swift   추가/수정 공용 폼 화면
```

- 화면/역할별로 파일을 나눠 각 파일이 한 가지 일만 하도록 한다
  (초보 단계에서 코드를 한눈에 파악하기 쉽게).

## 8. 성공 기준 (1단계 완료 조건)

- [ ] 앱을 실행하면 습관 목록 화면이 보인다. (습관이 없으면 안내 문구 표시)
- [ ] [+] 버튼으로 추가 화면을 열고, 이름·아이콘·색을 입력해 저장하면
      목록에 새 습관이 나타난다.
- [ ] 목록의 행을 탭하면 그 습관의 값이 채워진 수정 화면이 열리고,
      값을 바꿔 저장하면 목록에 반영된다.
- [ ] 행을 옆으로 밀어 삭제하면 목록에서 사라진다.
- [ ] 이름이 비어 있으면 저장 버튼이 비활성화(또는 저장 불가)된다.
- [ ] iOS 시뮬레이터에서 위 동작이 모두 정상 작동한다.

---

## 9. 향후 단계 (참고용, 이번 스펙 범위 아님)

- 2단계: `[Habit]`를 기기에 저장(영속화)해 앱 재실행 후에도 유지.
- 3단계: 날짜별 완료 기록 + 연속 달성일(streak) 계산·표시.
- 4단계: 습관 상세 화면, 달력 뷰, 주간 달성률 통계.
- 5단계: 매일 정해진 시각 로컬 알림(리마인더).
